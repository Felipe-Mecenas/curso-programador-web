# Colaboração no GitHub e Boas Práticas

Trabalhar em equipe exige organização para evitar conflitos de código.

## Processo de Trabalho (Workflow)
1. **Fork/Clone**: Obter uma cópia do repositório.
2. **Branch**: Criar uma ramificação para cada nova funcionalidade (evita mexer direto na `main`).
3. **Commit**: Salvar as alterações localmente com mensagens claras.
4. **Push**: Enviar as alterações para o servidor remoto.
5. **Pull Request (PR)**: Solicitar a integração do seu código ao projeto principal.

## Boas Práticas
* **Commits Atômicos**: Faça commits pequenos e frequentes.
* **Mensagens Descritivas**: Use verbos no imperativo (ex: "Adiciona tutorial de Git").
* **Code Review**: Sempre peça para um colega revisar seu PR antes do merge.
